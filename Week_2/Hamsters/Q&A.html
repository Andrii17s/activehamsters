Перший спосіб підключення стилей CSS називається “звязування” для даного способу використовуються зовнішні таблиці стилей. Cтворюємо новий документ, зберігаємо цей документ як style.css, створюємо папку з назвою сss і в дану папку поміщається файл style.css, в папці сss нам буде потрібно розміщувати всі файли з розширенням сss. Наш файл style.css повинен бути в даній папці. Наступні дії, тепер нам буде потрібно цей файл підключити до HTML документу.
Відкриваємо наш HTML документ за допомогою текстового редактора, в області head підключається даний стиль CSS, за допомогою спеціального тегу який називається “link”, це є одинарний тег, в цей тег додається декілька атрибутів.
Перший атрибут “href” Цей атрибут буде вказувати браузеру де знаходяться файл style.css він знаходиться в папці CSS файл style.css

отже перед закриваючим тегом </head> прописуємо тег “link” <link до тегу “link” додаємо атрибут “href” в даному атрибуті виставляємо дорівнює де в лапках вказуємо спочатку де знаходиться папка CSS потім ставим слеш і назву файлу style.css який знаходиться в даній папці.
Другий атрибут який вказується після атрибуту “href” це атрибут “type” Повинні вказати тип підключаємого документу, після href=”css/style.css” прописуємо атрибут “type” далі дорівнює і в лапках вказуємо тип документу, text далі слеш і css
Наступний атрибут це атрибут “rel” означає “взаємозвязок” далі в нас дорівнює, а в лапках значення “stylesheet”.

Не має значення який з атрибутів “type” чи “rel” ви пропишите першим після атрибуту “href”		 <link href=”css/style.css” rel=”stylesheet” type=”text/css” >

я розумію чому між стилями можуть бути конфлікти при різних варіантах підключення
Пріоритет способу підключення стилів CSS.
На практиці, коли ви будете працювати з реальними веб-сторінками, дуже часто виникає ситуація, що одному і тому ж елементу на веб-сторінці може бути призначено один і той самий стиль з різними значеннями, але з різних джерел.
Який з цих стилів буде пріоритетнішим для браузера? Який він відобразить в результаті?
Давайте розглянемо таку ситуацію. У нас є елемент абзацу. Давайте зараз застосуємо до цього абзацу різні стилі CSS, але з різних джерел і подивимося, який стиль в результаті буде застосований для елемента.
Файл index.html: <style>
.red {
color:blue;
}
</style>
...
<p class="red" style="color:green;">Абзац</p>
Файл style.css <style>
.red {
color:red;
}
</style>
В даному прикладі у нас є 3 джерела, в яких задається значення кольору тексту для елементу абзацу з атрибутом class="red" : 1 - атрибут style
2 - усередині елементу style
3 - усередині файлу style.css
Найголовніше, значення кольору тексту, в кожному з цих джерел суперечить один одному.
Якщо ви відкриєте цю веб-сторінку у браузері, ви побачите, що абзац відобразитися зеленим кольором тексту.
Т.е. найпріоритетнішим з точки зору підключення стилів CSS являтиметься внутрішній стиль, який застосовується за допомогою атрибуту style.
Якщо видалити цей стиль, то пріоритетнішим стає стиль усередині елементу style. Це є другим по пріоритету способом застосування стилів CSS.
Останнім по пріоритету застосування стилів CSS являтимуться стилі усередині файлу style.css.
Сенс тут в тому, що стилі CSS, які знаходяться щонайближче до елементу і будуть найбільш пріорітетними.

Програміст може визначити правило стилю на основі ідентифікатора елементів. Всі елементи з однаковим ідентифікатором будуть застосовані з визначеним стилем. Щоб вибрати елемент з певним ідентифікатором, має бути символ # (хеш), за яким слід ідентифікувати елемент.
Припустимо, що HTML-файл містить оператор наступним чином. <h1 id="header1">Hellow World</h1>
Файл CSS містить наступний фрагмент коду. #header1 {

color: blue;

}
Коли браузер інтерпретує вираз HTML, він перевіряє id елемента h1, який є header1 у файлі CSS. Потім він застосовує визначене правило CSS до елемента h1. Тому текст Hello World з'явиться у синьому кольорі.
Ідентифікатор є унікальним на сторінці. Таким чином, селектор ідентифікаторів використовується для одного унікального елемента.
Подібно до ідентифікатора, програміст може визначати правила стилю на основі класу елемента. Всі елементи з одним класом будуть застосовані з певним стилем. Щоб вибрати елемент з певним класом, має бути . символ (період), за яким слідує назва класу.

Припустимо, що HTML-файл містить наступні оператори.		 <h1 id="header1">Hellow World 1</h1>
<h1 id="header1">Hellow World 2</h1>
Файл CSS містить наступний фрагмент коду. .header1 {

color: blue;

}
Коли браузер інтерпретує вираз HTML, він перевіряє клас елементу h1, який є header1 у файлі CSS. Потім він застосовує визначене правило CSS до всіх елементів h1. Тому текст Hello World 1 і Hello World 2 відображатимуться у синьому кольорі.
Можна використовувати один і той же клас на декількох елементах. Тому селектор класу використовується для декількох елементів.
id - селектор в CSS, який стилює елемент з заданим id, тоді як class - селектор в CSS, який стилює вибрані елементи за допомогою заданого класу.
Синтаксис ідентифікатора - це #id {css декларації; }. Синтаксис класу - це декларації .class {css; }
Крім того, використання ідентифікатора полягає у застосуванні стилізації до одного конкретного елемента. Використання класу - це застосування стилів до декількох елементів.
Ідентифікатор і клас - два селектори в CSS, що дозволяє застосовувати стилі до елементів HTML. Основною відмінністю між ідентифікатором та класом у CSS є те, що id використовується для застосування стилізації до одного унікального елемента, а клас використовується для застосування стилізації до декількох елементів.

По-перше максимальне використання CSS дає можливість істотно скоротити обсяг HTML-розмітки. Зрозуміло, при цьому зростає обсяг CSS-коду, але CSS-файл, будучи зовнішнім по відношенню до групи HTML-документів, як правило, кешируєтся браузером і не вимагає повторного завантаження з кожним документом, пов'язаним з цим файлом. Зменшення обсягу HTML-коду позитивно позначається на швидкості завантаження веб-сторінок. Крім того, зменшення відношення обсягу службової розмітки до обсягу корисного змісту вельми благотворно позначається на ранжируванні сторінки у видачі пошукових систем.
По-друге реалізація принципу поділу змісту та подання на рівні кінцевого коду веб-сторінок полегшує можливу подальшу автоматичну обробку вмісту цих сторінок.
По-третє CSS дозволяє управляти поданням даних, з огляду на специфіку середовища або пристрої виведення інформації, включаючи, наприклад, програми синтезу мови, дисплеї Брайля і т. Д. CSS вміє працювати з ними в режимі, абсолютно прозорому для користувача. За умови, наприклад, що розробником передбачені індивідуальні листи стилів (або правила в загальному листі стилів) для подання вмісту сторінок сайту не тільки на екрані настільного комп'ютера або ноутбука, але і на екрані кишенькового комп'ютера, а також на аркуші паперу під час друку, користувачеві немає ніякої потреби шукати на сторінці кнопки або посилання а-ля «Версія для КПК» або «Версія для друку». Ні про що не замислюючись, користувач може сміливо зайти на сторінку з тим же самим URL з «надолонника», побачивши її в спеціально пристосованому оформленні. Точно так же користувач може стандартними засобами браузера відправити необхідну сторінку на друк і отримати її «тверду копію» без зайвого оформлення, наприклад, без неактуальних в даному контексті навігаційної панелі і форми підписки на новини.
По-четверте можливості CSS з управління візуальними уявленнями документів значно багатшими, ніж аналогічні можливості застарілих тегів і атрибутів візуального форматування, передбачених в HTML. CSS, фактично, крім того, що здатний повністю замінити всі без винятку візуальні елементи HTML, надає величезну кількість можливостей по управлінню візуальним представленням, яких ніколи не було в HTML. Іншими словами, CSS дозволяє контролювати будь-які аспекти візуального представлення документів з точністю до останнього пікселя.
По-п'яте прогресивні технології краще для застарілих браузерів. Бо принцип поділу змісту і уявлення гарантує текстуальную доступність контенту по всіх усюдах. Судіть самі - якщо раптом агент користувача з якихось причин не підтримує CSS (або відвідувач сайту самостійно заборонив CSS в своєму браузері), невелика біда. HTML-код сторінки буде в будь-якому випадку оброблений. Графічний браузер відобразить таку сторінку з оформленням, прийнятим за замовчуванням. Що стосується сторінок, «напханих» візуальними позначками та атрибутами HTML, відображення сайту в старих браузерах неможливо дати ніяких гарантій.

Опис CSS атрибут position встановлює спосіб позиціонування елемента щодо вікна браузера або інших об'єктів на веб-сторінці.
Синтаксис position: absolute | fixed | relative | static | inherit | sticky
Значення Вказує, що елемент абсолютно позиціонується, при цьому інші елементи відображаються на веб-сторінці немов абсолютно позиціонованого елемента і немає. Положення елемента задається властивостями left, top, right і bottom, також на положення впливає значення властивості position батьківського елемента. Так, якщо у батька значення position встановлено як static або батька немає, то відлік координат ведеться від краю вікна браузера. Якщо у батька значення position задано як fixed, relative або absolute, то відлік координат ведеться від краю батьківського елементу. absolute
За своєю дією це значення близьке до absolute, але на відміну від нього прив'язується до зазначеної властивостями left, top, right і bottom точці на екрані і не змінює свого положення при прокручуванні веб-сторінки. Браузер Firefox взагалі не відображає смуги прокрутки, якщо положення елемента задано фіксованим, і воно не поміщається цілком у вікно браузера. У браузері Opera хоча і показуються смуги прокрутки, але вони ніяк не впливають на позицію елемента. fixed
Положення елемента встановлюється щодо його вихідного місця. Додавання властивостей left, top, right і bottom змінює позицію елемента і зрушує його в ту чи іншу сторону від первісного розташування. relative
Елементи відображаються як зазвичай. Використання властивостей left, top, right і bottom не призводить до якихось результатів. static
Успадковує значення батька. inherit
Липке або приклеюване позиціонування використовується для фіксації елемента у заданомоу місці при прокручуванні сторінки. Як правило, застосовується до заголовків, меню або навігації для швидкого доступу або наочності. sticky
Треба розуміти, що при вертикальному прокручуванні для елемента необхідно завдавати тільки top та bottom. При горизонтальному прокручуванні будуть працювати тільки властивості left та right.
Також треба враховувати, що висота або ширина повинні бути більшими за сам елемент, бо в інших випадках не з'явиться стрічка прокрутки та елемент лишатиметься на місці.
