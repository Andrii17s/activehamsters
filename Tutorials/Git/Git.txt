

                                    КОМАНДИ

git pull - синхронізація стану локального гіту з віддаленим

git status - перегляд місця поточного перебування (гілки) та стан її файлів

git commit [commit_name] - додати коміт

                                   ГІЛКИ

git branch [branch_name] - створини нову гілку

git checkout -b [branch_name] - створити та перейти в нову гілку

git checkout [branch_name] - перейти в гілку


                                   ЗЛИТТЯ ГІЛОК

git merge [branch_name] - злиття гілок (робити пепебуваючи в с main)

git rebase main - злиття гілок з дочірньої гілки
                                 
                                   
                                   ПЕРЕМІЩЕННЯ ПО ДЕРЕВУ ГІЛОК (ВІДНОСНІ ССИЛКИ)


git checkout [commit_hash] - перехід до конкретного коміту в гілці

git checkout [branch_name]^ ([branch_name]^^, [branch_name]^^^...) - перехід на гілку и нащадка коміта

git checkout HEAD^ (HEAD^^, HEAD^^...) - перехід на нащадка коміта в гілці

git checkout HEAD~4 (HEAD~5, HEAD~10...) - перехід на  N-нащадка коміта в гілці

git branch -f main HEAD~3 - переміщує гілку main  (force) на 3 нащадка після HEAD 

                                  
                                  ВІДМІНА ЗМІН 

git reset HEAD~1 (HEAD~5, HEAD~10...) - переміщує гілку на попередній коміт, останніх наче й не було (використовується локально)

git revert HEAD~1 (HEAD~5, HEAD~10...) - додає коміт з змінами оберненими попередньому (використовується при роб. з віддаленим гітом, можна робити push)

                                      
                                  ПЕРЕНОС ЗМІН

git cherry-pick [commit_hash1] [commit_hash2] [commit_hash3] - копіювання комітів в main (бути в main, коміти з довільної гілки)

git rebase -i HEAD~4 - переміщення (пропуск) комітів в потрібній послідовності
